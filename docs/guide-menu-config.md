# 菜单配置操作手册

## 1. 功能概述

菜单配置功能允许系统管理员自定义系统的导航菜单结构，包括添加、编辑、删除菜单项，设置菜单的权限控制，以及调整菜单的显示顺序。

## 2. 访问路径

1. 登录系统后，点击左侧导航栏中的 **系统管理**
2. 在系统管理下拉菜单中，点击 **菜单配置**

## 3. 界面介绍

菜单配置页面包含以下元素：

- **添加菜单项** 按钮：用于添加新的菜单项
- **菜单树**：显示当前系统的菜单结构，支持展开/折叠
- **操作按钮**：每个菜单项右侧有 "编辑" 和 "删除" 按钮
- **保存配置** 按钮：用于保存当前的菜单配置
- **菜单顺序号**：用于控制菜单项的显示顺序，数字越小显示越靠前

## 4. 操作指南

### 4.1 添加菜单项

1. 点击 **添加菜单项** 按钮，打开添加菜单项对话框
2. 填写以下信息：
   - **菜单路径**：菜单项的路由路径，必须以 `/` 开头，只能包含字母、数字、 `-` 和 `/`
   - **菜单名称**：菜单项的显示名称
   - **图标**：菜单项的图标，使用 Element Plus 的图标名称，如 `el-icon-s-home`
   - **菜单顺序**：控制菜单项的显示顺序，数字越小显示越靠前，默认为 999
   - **角色权限**：选择可以访问该菜单项的角色，可选
   - **父菜单**：选择该菜单项的父菜单，若不选择则作为一级菜单项
3. 点击 **保存** 按钮，完成添加

### 4.2 编辑菜单项

1. 在菜单树中，找到需要编辑的菜单项
2. 点击菜单项右侧的 **编辑** 按钮，打开编辑菜单项对话框
3. 修改菜单项的信息，包括菜单顺序号
4. 点击 **保存** 按钮，完成编辑

### 4.3 删除菜单项

1. 在菜单树中，找到需要删除的菜单项
2. 点击菜单项右侧的 **删除** 按钮
3. 在弹出的确认对话框中，点击 **确定** 按钮，完成删除

### 4.4 保存配置

1. 完成菜单的添加、编辑、删除操作后，点击页面底部的 **保存配置** 按钮
2. 系统会保存当前的菜单配置

## 5. 菜单结构说明

### 5.1 菜单层次结构

系统菜单分为三个层次：

1. **一级菜单项**：直接显示在左侧导航栏中，如 "首页"、"公告管理"、"消息中心" 等
2. **二级菜单项**：显示在一级菜单项的下拉菜单中，如 "公告管理" 下的 "公告列表" 和 "发布公告"

### 5.2 菜单顺序

菜单项的显示顺序由 `order` 属性控制：
- 数字越小，显示位置越靠前
- 默认为 999，未设置顺序的菜单项会显示在最后
- 同级菜单项按顺序号从小到大排列

### 5.3 菜单配置分类

系统菜单配置分为三个独立的配置文件：

1. **普通用户菜单**：所有登录用户可见
2. **管理员菜单**：只有管理员角色可见
3. **未登录用户菜单**：未登录用户可见

## 6. 权限控制

通过设置菜单项的 **角色权限**，可以控制不同角色的用户能够看到和访问的菜单项：

- **管理员**：可以看到所有菜单项
- **普通用户**：只能看到没有设置角色权限或设置了 "普通用户" 权限的菜单项

## 7. 图标说明

系统使用 Element Plus 的图标库，常用图标名称如下：

- `el-icon-s-home`：首页
- `el-icon-message`：消息
- `el-icon-bell`：通知
- `el-icon-user`：用户
- `el-icon-setting`：设置
- `el-icon-document`：文档
- `el-icon-folder`：文件夹
- `el-icon-plus`：添加
- `el-icon-edit`：编辑
- `el-icon-delete`：删除

更多图标请参考 [Element Plus 图标文档](https://element-plus.org/zh-CN/component/icon.html)

## 8. 注意事项

1. **菜单路径必须唯一**：每个菜单项的路径必须唯一，不能重复
2. **路径格式**：路径必须以 `/` 开头，只能包含字母、数字、 `-` 和 `/`
3. **菜单顺序**：合理设置菜单顺序号，确保菜单显示逻辑清晰
4. **权限继承**：子菜单项会继承父菜单项的权限设置
5. **保存配置**：所有菜单修改后，需要点击 "保存配置" 按钮才能生效
6. **前端路由**：添加新菜单项时，需要确保对应的前端路由已经定义

## 9. 故障排除

### 9.1 菜单项不显示

- 检查菜单项的权限设置，确保当前用户角色有访问权限
- 检查菜单项的路径是否正确
- 检查是否点击了 "保存配置" 按钮

### 9.2 菜单项点击无响应

- 确保对应的前端路由已经定义
- 检查菜单项的路径是否与前端路由路径一致

## 10. 示例

### 10.1 添加一级菜单项

1. 点击 "添加菜单项" 按钮
2. 填写：
   - 菜单路径：`/new-feature`
   - 菜单名称：`新功能`
   - 图标：`el-icon-document`
   - 菜单顺序：`3`（显示在 "消息中心" 之后）
   - 角色权限：选择 "管理员" 和 "普通用户"
   - 父菜单：选择 "无（作为一级菜单）"
3. 点击 "保存" 按钮
4. 点击 "保存配置" 按钮

### 10.2 添加二级菜单项

1. 点击 "添加菜单项" 按钮
2. 填写：
   - 菜单路径：`/new-feature/detail`
   - 菜单名称：`功能详情`
   - 图标：`el-icon-edit`
   - 菜单顺序：`1`（显示在二级菜单的第一个位置）
   - 角色权限：选择 "管理员"
   - 父菜单：选择 "新功能"
3. 点击 "保存" 按钮
4. 点击 "保存配置" 按钮
